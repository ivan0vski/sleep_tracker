# Что происходит при создании нового плана

## 1. Точка входа: Settings → SetupWizard

Настройки → кнопка «Новый план» → `prompt("новый план")` → `SetupWizard.open()` с callback `onComplete: () => App.refreshPlan()`.

## 2. Визард собирает конфиг (7 шагов)

- `currentWake` — текущий подъём (напр. 09:00)
- `targetWake` — целевой подъём (напр. 06:00)
- `stepMinutes` — сдвиг за фазу (15/20/30/45/60 мин)
- `phaseDays` — дней на фазу (1-7, по умолчанию 7)
- `desiredSleepHours` — желаемый сон (7/7.5/8/8.5/9 ч)
- `routineSteps` — шаги вечернего распорядка (дефолтные 7 шагов, можно отредактировать)

## 3. createPlan() — что записывается в БД

1. **Архивация старого плана** — `DB.getActivePlan()` → если есть, `DB.updatePlanStatus(id, 'archived')`
2. **Расчёт фаз** — `PhaseEngine.calculatePhases(config)` → массив фаз, каждая с `wake`, `bed`, `startDate`, `endDate`, `color`
3. **Сохранение плана** — `DB.savePlan(plan)` → в IndexedDB store `sleepPlans`
4. **Сохранение распорядка** — `DB.saveRoutineSteps(routineSteps)` → store `routineSteps` (clear + put)
5. **Callback** — `onComplete(plan)` → `App.refreshPlan()`

## 4. App.refreshPlan() — что обновляется в UI

```
loadActivePlan()           → читает план из БД в activePlan
  ↓
SleepForm.setPlan(plan)    → form.js запоминает план
Protocol.setPlan(plan)     → protocol.js запоминает план
Routine.setPlan(plan)      → routine.js запоминает план
Instruction.setPlan(plan)  → instruction.js запоминает план
renderPhaseBar()           → перерисовка прогрессбара
renderPhaseLabel()         → перерисовка подписи фазы
```

## 5. Куда план попадает и что меняет

| Модуль | Что меняется от плана |
|---|---|
| **app.js (шапка)** | Прогрессбар фаз — квадратики текущей фазы с ✓/✕. Подпись «Фаза N · день M · подъём HH:MM · отбой HH:MM» |
| **form.js (трекер)** | Индикаторы ✓/✕ у полей «лёг» и «проснулся» — сравнивает введённое время с `phase.bed` и `phase.wake` (±15 мин) |
| **protocol.js** | Все времена динамические: кофеин до X, тренировки до Y, экраны выключить к Z — рассчитываются от `wake`/`bed` текущей фазы через `PhaseEngine.getDayContext()` |
| **routine.js (распорядок)** | Времена шагов рассчитываются от `bed` текущей фазы. Шаг «Подготовка к ужину» — bed-160 мин, «Экраны» — bed-120 и т.д. |
| **instruction.js** | Два динамических времени — первый шаг распорядка и screensOff — пересчитываются от фазы |
| **history.js** | Бейдж «фаза N день M» (раскрашен цветом фазы) + индикатор попадания ✓/✕ для каждой записи |
| **phaseCalendar.js** | Дни в календаре раскрашены цветом фазы |
| **settings.js** | Кнопка «Текущий план» → детальный экран со статистикой, кнопка «Сбросить план» становится видна |

## 6. Автозавершение

`checkPlanCompletion()` в `App.init()` — если сегодня > последний день последней фазы → план автоматически получает статус `'completed'`, все модули получают `setPlan(null)`, показывается модалка «План завершён!».

## Итог

План — центральный объект, от которого зависят времена во всех 5 вкладках приложения. Он определяет wake/bed для каждого дня, а от wake/bed считаются все протокольные времена, распорядок, индикаторы попадания и визуальная раскраска.
